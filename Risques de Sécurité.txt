Risques de Sécurité:
DNS spoofing:
	Le DNS Spoofing (Domain Name Service Spoofing ou usurpation de DNS en français) consiste à empoisonner les entrées d’un serveur DNS pour rediriger un utilisateur ciblé vers un site Web malveillant contrôlé par l’attaquant.(https://www.proofpoint.com/fr/threat-reference/dns-spoofing)

SQL Injection:
	SQL injection usually occurs when you ask a user for input, like their username/userid, and instead of a name/id, the user gives you an SQL statement that you will unknowingly run on your database.(https://www.w3schools.com/sql/sql_injection.asp)

DOS:
	Une attaque par déni de service (abr. DoS attack pour Denial of Service attack en anglais) est une attaque informatique ayant pour but de rendre indisponible un service, d'empêcher les utilisateurs légitimes d'un service de l'utiliser.(19/11/2024 https://fr.wikipedia.org/wiki/Attaque_par_d%C3%A9ni_de_service)
Denial of service is typically accomplished by flooding the targeted machine or resource with superfluous requests in an attempt to overload systems and prevent some or all legitimate requests from being fulfilled.[1] The range of attacks varies widely, spanning from inundating a server with millions of requests to slow its performance, overwhelming a server with a substantial amount of invalid data, to submitting requests with an illegitimate IP address.(19/11/2024 https://en.wikipedia.org/wiki/Denial-of-service_attack)

les lois de sécurité applicables:
RGPD:
1.Minimisation des données : Les données personnelles doivent être limitées à celles qui sont nécessaires pour atteindre les finalités du traitement.
2.Responsabilité : Les entreprises sont responsables du traitement des données personnelles et doivent être en mesure de démontrer leur conformité au RGPD.
3.Sécurité : Les entreprises doivent prendre des mesures appropriées pour garantir la sécurité et la confidentialité des données personnelles.
